# 🚀 پنل 3x-ui روی Railway با ذخیره‌سازی دائمی

این پروژه، پنل مدیریت **3x-ui (سنایی)** را روی سرویس ابری **Railway** اجرا میکند. با استفاده از این پروژه، میتوانید یک پنل مدیریت VPN کامل را بدون نیاز به سرور مجازی (VPS) راهاندازی کنید.

---

## ✨ ویژگیها

- ✅ **اجرا روی Railway** – بدون نیاز به سرور مجازی
- ✅ **ذخیرهسازی دائمی** – با استفاده از Volume، اطلاعات پنل پس از ریاستارت از بین نمیرود
- ✅ **نسخه 3.5.0** – آخرین نسخه پنل سنایی
- ✅ **Fail2ban** – امنیت بیشتر با مسدودسازی IPهای متخلف
- ✅ **اصلاح خودکار لینکها** – تبدیل لینکها به فرمت مناسب Railway (پورت 443 + TLS)
- ✅ **ساباسکریپشن خام** – بدون صفحه گرافیکی، فقط کانفیگهای خام
- ✅ **پشتیبانی از VLESS+WebSocket** – تنظیمات بهینه برای Railway

## 🚀 راهاندازی در Railway

### ۱. آمادهسازی گیتهاب

1. یک ریپازیتوری جدید در گیتهاب بسازید.
2. فایلهای زیر را در آن آپلود کنید:
   - `Dockerfile`
   - `start.sh`
   - `nginx.conf.template` (اختیاری)

### ۲. دیپلوی در Railway

1. وارد [Railway.app](https://railway.app) شوید.
2. روی **New Project** → **Deploy from GitHub repo** کلیک کنید.
3. ریپازیتوری خود را انتخاب کنید.
4. Railway بهطور خودکار شروع به بیلد و دیپلوی میکند.

### ۳. تنظیمات پس از دیپلوی

#### تنظیم Target Port
- به **Settings → Networking** بروید.
- **Target Port** را روی **3000** تنظیم کنید.

#### ایجاد Volume برای ذخیرهسازی دائمی
- به **Settings → Volumes** بروید.
- یک Volume جدید با این تنظیمات بسازید:
  - **Name:** `sanaei-data`
  - **Mount Path:** `/etc/x-ui`
- روی **Create** کلیک کنید و سپس **Redeploy** انجام دهید.

#### تنظیم متغیرهای محیطی (اختیاری)
- به **Settings → Variables** بروید.
- متغیرهای زیر را در صورت نیاز اضافه کنید:

| متغیر | مقدار پیشفرض | توضیح |
|-------|---------------|--------|
| `NGINX_PORT` | `3000` | پورت ورودی Nginx |
| `PANEL_PORT` | `2053` | پورت داخلی پنل |
| `XUI_VERSION` | `v3.5.0` | نسخه پنل سنایی |

## 🔧 تنظیمات اینباند در پنل

بعد از ورود به پنل، یک اینباند با این تنظیمات بسازید:

| تنظیم | مقدار |
|-------|-------|
| **Protocol** | `VLESS` |
| **Port** | `8080` |
| **Network** | `ws` (WebSocket) |
| **Path** | `/ws` (یا هر مسیر دلخواه) |
| **Security** | `none` (TLS در سطح Nginx اعمال میشود) |

### لینک اتصال نهایی

لینکی که از پنل دریافت میکنید به این شکل است:

```
vless://UUID@دامنه.up.railway.app:8080?encryption=none&host=دامنه.up.railway.app&path=%2Fws&security=none&type=ws#-نام
```

**اما** به دلیل تنظیمات `sub_filter` در Nginx، لینک به صورت خودکار به این فرمت تبدیل میشود:

```
vless://UUID@دامنه.up.railway.app:443?encryption=none&host=دامنه.up.railway.app&path=%2Fws&security=tls&sni=دامنه.up.railway.app&fp=chrome&insecure=0&allowInsecure=0&type=ws#نام
```

## 🛠️ عیبیابی

### ۱. پنل بالا نمیآید
- **راهحل:** لاگهای Railway را بررسی کنید. مطمئن شوید که `Target Port` روی `3000` تنظیم شده است.

### ۲. اطلاعات پس از ریاستارت از بین میرود
- **راهحل:** مطمئن شوید که Volume به درستی به مسیر `/etc/x-ui` متصل شده است.

### ۳. لینک ساب کار نمیکند
- **راهحل:** مطمئن شوید که `sub_filter` در `start.sh` به درستی تنظیم شده است و دامنه شما با `$host` جایگزین میشود.

### ۴. خطای 502 Bad Gateway
- **راهحل:** بررسی کنید که پنل سنایی روی پورت `2053` در حال اجرا است. در کنسول Railway دستور زیر را اجرا کنید:
  
  ```bash
  curl http://127.0.0.1:2053/managepanel/
  ```

## 📝 نکات مهم

- **ذخیرهسازی**: برای حفظ اطلاعات، حتماً Volume به مسیر `/etc/x-ui` متصل شود.
- **امنیت**: رمز عبور پیشفرض را بلافاصله پس از اولین ورود تغییر دهید.
- **آپدیت**: برای آپدیت به نسخه جدید، کافی است `XUI_VERSION` را در `Dockerfile` تغییر دهید.
